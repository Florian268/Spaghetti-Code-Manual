UML Diagrams

What Are UML Diagrams

UML, or Unified Modeling Language, diagrams model and document software. There exists several different types of these diagrams. In this section, we will go over some of the most popular types of diagrams and how you can implement them into your coding endevors.

How They Help Spaghetti Code

UML diagrams help visualize the more difficult and abstract parts of coding. New programmers may find it difficult to keep all the architecture of their software organized, especially if it involves several types of different classes that interact. So, a visual representation allows programmers to keep all of their structure straight, as well as be able to share with others how their system works.

Class Diagrams

Perhaps the most popular is the Class Diagram. This displays boxes that label each class and its methods (marked by its privacy) and its connection to each other class.

Component Diagrams

This diagrams shows each component in a system and how they interact with each other.

Development Diagrams

Development diagrams show how software affects the physical components on a computer. These are crucial for showing runtimes for hardware.

Composite Structure Diagrams

Composite Structure Diagrams show exactly what each objects contains in a program. It provides a look at configuration classes, interfaces, packages, and the relationship between them.

Object Diagrams

Similar to a Class Diagram, an Object Diagram shows the relationship between different objects in each class. However, this is a more high-level representation than a Class Diagram, thus is most useful in a corporate environment, where most people won't understand the intricacies of software.
